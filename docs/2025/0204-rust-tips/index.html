VSCode

Cargo.toml がワークスペースのルートに無いときの設定

参考: RustのCargo.tomlの場所がrootに無い時にVSCodeでrust-analyzerのエラーを回避する方法

.vscode/settings.json に場所を記述する

{
  "rust-analyzer.linkedProjects": [
    "${workspaceFolder}/path/to/Cargo.toml"
  ]
}
